## Aufgabe 13 – Routing ohne Port Forwarding mit Ingress

In dieser Aufgabe wird der AI Service ohne `kubectl port-forward` erreichbar gemacht.

Bisher wurde der Zugriff lokal weitergeleitet:

```bash
kubectl port-forward service/ai-service 8001:8000
```

Nun wird ein **Ingress Controller** verwendet.

Ziel:

* Zugriff ohne Port Forwarding ermöglichen
* Ingress Controller installieren
* Routing über eine feste URL einrichten
* AI Service im Browser testen

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `13. Aufgabe` um.**

---

### Schritt 1 – Ingress Controller installieren

Installiert den NGINX Ingress Controller direkt per Manifest:

```bash
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/cloud/deploy.yaml
```

---

### Schritt 2 – Installation prüfen

```bash
kubectl get pods -n ingress-nginx
```

Wartet, bis der Controller läuft:

```text
ingress-nginx-controller-...   1/1   Running
```

---

### Schritt 3 – AI Service Service prüfen

Der AI Service bleibt ein normaler interner Kubernetes Service.

Öffnet:

```text
k8s/ai-service-service.yaml
```

Inhalt:

```yaml
apiVersion: v1
kind: Service
metadata:
  name: ai-service
spec:
  selector:
    app: ai-service
  ports:
    - port: 8000
      targetPort: 8000
```

Anwenden:

```bash
kubectl apply -f k8s/ai-service-service.yaml
```

---

### Schritt 4 – Ingress Datei erstellen

Erstellt im Ordner `k8s` eine neue Datei:

```text
k8s/ai-service-ingress.yaml
```

---

### Schritt 5 – Ingress konfigurieren

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: ai-service-ingress
spec:
  ingressClassName: nginx
  rules:
  - host: ai.localhost
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: ai-service
            port:
              number: 8000
```

---

### Schritt 6 – Ingress deployen

```bash
kubectl apply -f k8s/ai-service-ingress.yaml
```

---

### Schritt 7 – Ingress prüfen

```bash
kubectl get ingress
```

Erwartung:

```text
NAME                 CLASS   HOSTS          ADDRESS   PORTS   AGE
ai-service-ingress   nginx   ai.localhost   ...       80      ...
```

---

### Schritt 8 – AI Service testen

Im Browser:

```text
http://ai.localhost/analysis
```

Zusätzlich:

```text
http://ai.localhost/live
http://ai.localhost/ready
```

Falls `ai.localhost` nicht funktioniert:

```bash
curl -H "Host: ai.localhost" http://localhost/analysis
```

---
