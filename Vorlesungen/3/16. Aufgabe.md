## Aufgabe 16 – Deployment automatisieren mit Script

In dieser Aufgabe wird der bisher manuelle Ablauf automatisiert.

Bisher:

```text
docker compose build
kubectl apply -f k8s/
```

Diese Schritte werden nun in einem Script zusammengefasst.

Ziel:

* wiederkehrende Schritte automatisieren
* ein einheitliches Deployment-Script erstellen
* Deployment mit einem Befehl starten

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `16. Aufgabe` um.**

---

### Schritt 1 – Script erstellen

Erstellt im Projekt-Root eine Datei:

```text
deploy.sh
```

---

### Schritt 2 – Script Inhalt hinzufügen

```bash
#!/usr/bin/env bash

set -e

PROJECT_NAME=$(basename "$PWD" | tr '[:upper:]' '[:lower:]' | tr -d ' .' | tr '_' '-')

DATA_IMAGE="${PROJECT_NAME}-data-service:latest"
AI_IMAGE="${PROJECT_NAME}-ai-service:latest"

echo "---------------------------------"
echo "Projekt: $PROJECT_NAME"
echo "Data Image: $DATA_IMAGE"
echo "AI Image: $AI_IMAGE"
echo "---------------------------------"

echo "Baue Docker Images..."
docker compose build

echo "Passe Kubernetes Image-Namen an..."
sed -i.bak "s|image: .*data-service.*|image: ${DATA_IMAGE}|g" k8s/data-service-deployment.yaml
sed -i.bak "s|image: .*ai-service.*|image: ${AI_IMAGE}|g" k8s/ai-service-deployment.yaml

rm -f k8s/*.bak

echo "Wende Kubernetes Konfiguration an..."
kubectl apply -f k8s/

echo "Starte Deployments neu..."
kubectl rollout restart deployment data-service
kubectl rollout restart deployment ai-service

echo "Aktuelle Pods:"
kubectl get pods

echo "Deployment abgeschlossen."
```

---

### Schritt 3 – Script ausführbar machen (macOS)

```bash
chmod +x deploy.sh
```

---

### Schritt 4 – Script ausführen

macOS / Linux:

```bash
./deploy.sh
```

Windows (Git Bash oder WSL):

```bash
bash deploy.sh
```

---

### Schritt 5 – Ergebnis prüfen

```bash
kubectl get pods
```

Erwartung:

```text
data-service   Running
ai-service     Running
```

---

### Schritt 6 – Anwendung testen

```text
http://ai.localhost/analysis
```

---

### Hinweis

```text
Scripts sind ein erster Schritt in Richtung CI/CD (Continuous Deployment).
```

---

Wenn das Script erfolgreich läuft, kann die gesamte Anwendung mit einem einzigen Befehl deployt werden.
