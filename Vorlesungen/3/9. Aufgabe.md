## Aufgabe 9 – AI Service mit ChatGPT erweitern

Der AI Service ruft bisher Daten vom Data Service ab.
Nun wird der Service erweitert, sodass aus den strukturierten Daten automatisch verständliche Erkenntnisse erzeugt werden.

Ziel:

* OpenAI API in den AI Service integrieren
* Daten vom Data Service analysieren lassen
* automatische Insights erzeugen
* API-Key über `.env` Datei verwenden

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `9. Aufgabe` um.**

---

### Schritt 1 – requirements.txt erweitern

Öffnet die Datei:

```text
ai-service/requirements.txt
```

Ergänzt die Dependency `openai`:

```txt
fastapi
uvicorn
requests
openai
```

---

### Schritt 2 – AI Service erweitern

Öffnet die Datei:

```text
ai-service/main.py
```

Ersetzt den bisherigen Inhalt durch:

```python
from fastapi import FastAPI
import requests
import os
from openai import OpenAI

app = FastAPI()

client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

@app.get("/analysis")
def analyze():
    response = requests.get("http://data-service:8000/metrics")
    data = response.json()

    prompt = f"""
    Analysiere folgende Google-Trends-Daten:

    {data}

    Gib eine kurze datenbasierte Interpretation:
    - stärkste Begriffe
    - Trends
    - Auffälligkeiten
    - Unterschiede zwischen Begriffen

    Wichtig:
    - keine Fantasie
    - nur Aussagen auf Basis der Daten
    - kurz und verständlich
    """

    ai_response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "user", "content": prompt}
        ]
    )

    return {
        "data": data,
        "analysis": ai_response.choices[0].message.content
    }

@app.get("/health")
def health():
    return {"status": "ok"}
```

---

### Schritt 3 – `.env` Datei erstellen

Erstellt im Projekt-Root eine Datei mit dem Namen:

```text
.env
```

Inhalt:

```env
OPENAI_API_KEY=YOUR_API_KEY
```

Ersetzt `YOUR_API_KEY` durch euren eigenen API-Key.

---

### Schritt 4 – `.gitignore` erweitern

Die Datei `.env` darf nicht in GitHub hochgeladen werden.

Ergänzt daher in eurer `.gitignore`:

```text
.env
```

---

### Schritt 5 – docker-compose.yml erweitern

Öffnet die Datei:

```text
docker-compose.yml
```

Erweitert den `ai-service` um die Umgebungsvariable:

```yaml
services:
  data-service:
    build:
      context: .
      dockerfile: data-service/Dockerfile
    ports:
      - "8000:8000"

  ai-service:
    build:
      context: .
      dockerfile: ai-service/Dockerfile
    ports:
      - "8001:8000"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
```

Docker Compose liest die `.env` Datei automatisch ein.

---

### Schritt 6 – Anwendung neu starten

Startet beide Services neu:

```bash
docker compose up --build -d
```

---

### Schritt 7 – AI Analyse testen

Öffnet im Browser:

```bash
http://localhost:8001/analysis
```

---

### Beispiel erwartetes Ergebnis

```json
{
  "data": {
    "terms": [
      {
        "name": "python",
        "mean": 54.3,
        "peak": 87,
        "trend": "increasing"
      }
    ]
  },
  "analysis": "Python zeigt im Datensatz den stärksten positiven Trend. Der Peak-Wert weist auf eine deutliche Nachfrage-Spitze hin."
}
```

Die konkreten Werte und Texte hängen von eurem Datensatz ab.
