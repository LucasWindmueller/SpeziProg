## Aufgabe 15 – Konfiguration, Skalierung und Fehlertest

In dieser Aufgabe wird die Anwendung weiter produktionsnah erweitert.

Bisher:

* Routing über Ingress ✔️
* Secrets sicher gespeichert ✔️

Nun kommen zentrale Kubernetes-Konzepte hinzu:

* Konfiguration auslagern (ConfigMap)
* Skalierung (mehrere Pods)
* Verhalten bei Fehlern beobachten

Ziel:

* ConfigMap erstellen und verwenden
* Services auf mindestens 3 Replicas skalieren
* Lastverteilung beobachten
* Verhalten bei Ausfällen testen

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `15. Aufgabe` um.**

---

## Teil 1 – ConfigMap erstellen

### Schritt 1 – ConfigMap Datei erstellen

Erstellt:

```text
k8s/app-config.yaml
```

Inhalt:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  DATA_SERVICE_URL: http://data-service:8000/metrics
  OPENAI_MODEL: gpt-4o-mini
```

---

### Schritt 2 – ConfigMap anwenden

```bash
kubectl apply -f k8s/ai-service-config.yaml
```

---

### Schritt 3 – AI Service anpassen

Öffnet:

```text
ai-service/main.py
```

Ersetzt die feste URL:

```python
DATA_SERVICE_URL = "http://data-service:8000/metrics"
```

durch:

```python
DATA_SERVICE_URL = os.getenv("DATA_SERVICE_URL")
MODEL = os.getenv("OPENAI_MODEL")
```

Und im API-Call:

```python
model=MODEL
```

---

### Schritt 4 – Deployment anpassen

Öffnet:

```text
k8s/ai-service-deployment.yaml
```

Ergänzt unter `env`:

```yaml
env:
- name: OPENAI_API_KEY
  valueFrom:
    secretKeyRef:
      name: ai-api-secret
      key: OPENAI_API_KEY
- name: DATA_SERVICE_URL
  valueFrom:
    configMapKeyRef:
      name: ai-service-config
      key: DATA_SERVICE_URL
- name: OPENAI_MODEL
  valueFrom:
    configMapKeyRef:
      name: ai-service-config
      key: OPENAI_MODEL
```

---

### Schritt 5 – Deployments skalieren

Öffnet beide Dateien:

```text
k8s/data-service-deployment.yaml
k8s/ai-service-deployment.yaml
```

Ändert:

```yaml
replicas: 3
```

### Schritt 13 – `.env` Datei für Docker Compose anpassen

Durch die Einführung der ConfigMap werden Konfigurationswerte nicht mehr im Code gesetzt, sondern über Umgebungsvariablen gelesen.

Damit der AI Service auch weiterhin mit Docker Compose funktioniert, müssen diese Variablen in der `.env` Datei ergänzt werden.

Öffnet die Datei:

```text
.env
```

Ergänzt folgende Werte:

```env
OPENAI_API_KEY=YOUR_API_KEY
DATA_SERVICE_URL=http://data-service:8000/metrics
OPENAI_MODEL=gpt-4o-mini
```

---

### Schritt 14 – docker-compose.yml prüfen

Stellt sicher, dass die Variablen im AI Service verwendet werden:

```yaml
ai-service:
  build:
    context: .
    dockerfile: ai-service/Dockerfile
  ports:
    - "8001:8000"
  environment:
    - OPENAI_API_KEY=${OPENAI_API_KEY}
    - DATA_SERVICE_URL=${DATA_SERVICE_URL}
    - OPENAI_MODEL=${OPENAI_MODEL}
```

---

### Schritt 15 – Anwendung neu starten

```bash
docker compose up --build -d
```

---

### Hinweis

```text
.env wird für Docker Compose verwendet  
ConfigMap wird für Kubernetes verwendet
```

Beide enthalten ähnliche Werte, werden aber in unterschiedlichen Umgebungen genutzt.

---

## Merksätze

```text
ConfigMap = Konfiguration  
Secret = sensible Daten  
Replicas = Skalierung + Ausfallsicherheit  
Readiness = Traffic steuern  
Liveness = Neustart auslösen
```

---

Wenn alle Services mit mindestens 3 Replicas laufen und das System auf Ausfälle korrekt reagiert, ist die Anwendung produktionsnah aufgebaut.
