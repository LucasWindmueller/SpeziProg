## Aufgabe 8 – AI Service um Health Endpoint erweitern

Nachdem Data Service und AI Service gemeinsam über Docker Compose laufen, wird der AI Service nun um einen Health Endpoint erweitert.

Dieser Endpoint wird später in Kubernetes benötigt, damit Kubernetes prüfen kann, ob der Container läuft und bereit ist.

Ziel:

* zusätzlichen Endpoint `/health` im AI Service erstellen
* Health Endpoint lokal über Docker Compose testen
* AI Service für Kubernetes Probes vorbereiten

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `8. Aufgabe` um.**

---

### Schritt 1 – Health Endpoint hinzufügen

Öffnet die Datei:

```text
ai-service/main.py
```

Erweitert die Datei um folgenden Endpoint:

```python
@app.get("/health")
def health():
    return {"status": "ok"}
```

Die Datei enthält damit mindestens zwei Endpoints:

* `GET /analysis`
* `GET /health`

---

### Schritt 2 – Anwendung neu starten

Startet die Anwendung im Projekt-Root neu:

```bash
docker compose up --build -d
```

---

### Schritt 3 – Health Endpoint testen

Prüft im Browser:

```bash
http://localhost:8001/health
```

---

### Beispiel erwartetes Ergebnis

```json
{
  "status": "ok"
}
```

---

### Schritt 4 – Analysis Endpoint erneut testen

Prüft zusätzlich:

```bash
http://localhost:8001/analysis
```

---

### Beispiel erwartetes Ergebnis

```json
{
  "terms": [
    {
      "name": "coffee",
      "mean": 72.9,
      "peak": 100,
      "trend": "decreasing"
    }
  ]
}
```

---

Wenn beide Endpoints erreichbar sind, ist der AI Service für spätere Kubernetes Health Checks vorbereitet.

Zum Beenden:

```bash
docker compose down
```
