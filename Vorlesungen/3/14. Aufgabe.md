
## Aufgabe 14 – Secret sicher mit Sealed Secrets speichern

In dieser Aufgabe wird der API-Key nicht mehr als normales Kubernetes Secret verwendet, sondern sicher verschlüsselt gespeichert.

Normale Secrets sind nur Base64-kodiert und nicht sicher für GitHub.

Mit **Sealed Secrets** wird das Secret verschlüsselt und kann sicher im Repository gespeichert werden.

Ziel:

* Secret sicher verschlüsseln
* Sealed Secrets verwenden
* API-Key nicht im Klartext speichern
* sichere Alternative zu `.env` und Base64 verstehen

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `14. Aufgabe` um.**

---

### Schritt 1 – kubeseal installieren

macOS (Homebrew):

```bash
brew install kubeseal
```

Windows (Chocolatey):

```powershell
choco install kubeseal
```

---

### Schritt 2 – Sealed Secrets Controller installieren

```bash
kubectl apply -f https://github.com/bitnami-labs/sealed-secrets/releases/latest/download/controller.yaml
```

---

### Schritt 3 – Installation prüfen

```bash
kubectl get pods -A | grep sealed
```

Erwartung:

```text
sealed-secrets-controller-...   Running
```

---

### Schritt 4 – temporäres Secret erstellen

Erstellt ein normales Kubernetes Secret als YAML (nur lokal!):

```bash
kubectl create secret generic ai-api-secret \
  --from-literal=OPENAI_API_KEY=sk-proj-gp3A... \
  --dry-run=client -o yaml > k8s/ai-api-secret.yaml
```

Wichtig:

Diese Datei enthält den API-Key (Base64-kodiert) und darf nicht in GitHub gespeichert werden.

---

### Schritt 5 – Secret verschlüsseln

```bash
kubeseal \
  --format yaml \
  < k8s/ai-api-secret.yaml \
  > k8s/ai-api-sealed-secret.yaml
```

---

### Schritt 6 – unverschlüsseltes Secret löschen

```bash
rm k8s/ai-api-secret.yaml
```

---

### Schritt 7 – .gitignore erweitern

Ergänzt:

```text
.env
ai-api-secret.yaml
```

---

### Schritt 8 – Sealed Secret deployen

```bash
kubectl delete -f k8s/
kubectl apply -f k8s/
```

---

### Schritt 9 – Secret prüfen

```bash
kubectl get secret ai-api-secret
```

Erwartetes Ergebnis:
```text
NAME            TYPE     DATA   AGE
ai-api-secret   Opaque   1      29m
```

Wenn das Secret existiert, wurde es automatisch vom Controller entschlüsselt.

---

### Schritt 10 – Anwendung testen

Test im Browser:

```text
http://ai.localhost/analysis
```

---

### Merksatz

```text
Base64 = Encoding  
SealedSecret = echte Verschlüsselung
```
