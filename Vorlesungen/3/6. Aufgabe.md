## Aufgabe 6 – AI Service erstellen und Data Service anbinden

Der AI Service ruft Daten über HTTP vom Data Service ab und stellt diese über einen eigenen Endpoint bereit.

Ziel:

* neuen Service erstellen
* Verbindung zum Data Service herstellen
* Daten über HTTP abrufen
* eigenen Endpoint `/analysis` bereitstellen

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `6. Aufgabe` um.**

---

### Schritt 1 – Projektstruktur erstellen

Erstellt im Projekt einen neuen Ordner für den AI Service:

```text
ai-service/
├── main.py
├── requirements.txt
```

---

### Schritt 2 – Dependencies definieren

Erstellt die Datei `requirements.txt` mit folgendem Inhalt:

```txt
fastapi
uvicorn
requests
```

---

### Schritt 3 – AI Service implementieren

Erstellt die Datei `main.py` mit folgendem Inhalt:

```python
from fastapi import FastAPI
import requests

app = FastAPI()

@app.get("/analysis")
def analyze():
    response = requests.get("http://localhost:8000/metrics")
    return response.json()
```

---

### Schritt 4 – Service lokal starten

Startet den AI Service im Ordner `ai-service` mit:

```bash
uvicorn main:app --reload --port 8001
```

---

### Schritt 5 – Endpoint testen

Öffnet im Browser:

```bash
http://127.0.0.1:8001/analysis
```

Hinweis:
Der Endpoint funktioniert nur, wenn der Data Service läuft und erreichbar ist.

---

### Beispiel erwartetes Ergebnis

```json
{
  "terms": [
    {
      "name": "coffee",
      "mean": 72.9,
      "peak": 100,
      "trend": "decreasing"
    }
  ]
}
```

---

Wenn der Endpoint Daten zurückgibt, ist der AI Service erfolgreich mit dem Data Service verbunden.
