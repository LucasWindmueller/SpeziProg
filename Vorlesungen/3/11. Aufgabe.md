## Aufgabe 11 – Kubernetes Deployment testen und debuggen

Nachdem die Kubernetes-Ressourcen erstellt wurden, wird geprüft, ob beide Services korrekt laufen und miteinander kommunizieren.

Ziel:

* Pods prüfen
* Services prüfen
* Logs auslesen
* Fehler systematisch debuggen
* Anwendung über Port Forwarding testen

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `11. Aufgabe` um.**

---

### Schritt 1 – Pods prüfen

```bash
kubectl get pods
```

Erwartung:

```text
NAME                            READY   STATUS    RESTARTS   AGE
ai-service-55cd9dc5b5-5cq6d     1/1     Running   0          7m10s
data-service-57b586bf47-bcblj   1/1     Running   0          18m
```

---

### Schritt 2 – Services prüfen

---

### Schritt 3 – Details eines Pods anzeigen

Falls ein Pod nicht startet:

```bash
kubectl describe pod POD_NAME
```

Beispiel:

```bash
kubectl describe pod ai-service-xxxxx
```

Achtet besonders auf den Bereich:

```text
Events
```

---

### Schritt 4 – Logs anzeigen

```bash
kubectl logs POD_NAME
```

Beispiel:

```bash
kubectl logs ai-service-xxxxx
```

---

### Schritt 5 – AI Service lokal erreichbar machen

```bash
kubectl port-forward service/ai-service 8001:8000
```

### Schritt 8 – Typische Fehler erkennen

| Fehler                       | Mögliche Ursache                            |
| ---------------------------- | ------------------------------------------- |
| `CreateContainerConfigError` | Secret fehlt oder falscher Key              |
| `ImagePullBackOff`           | Image nicht gefunden                        |
| `CrashLoopBackOff`           | App startet, stürzt aber ab                 |
| `Connection refused`         | falscher Port oder Service nicht erreichbar |
| `401 Unauthorized`           | API-Key falsch oder fehlt                   |
