
## Aufgabe 10 – Kubernetes Deployment erstellen

Nachdem Data Service und AI Service über Docker Compose laufen, wird die Anwendung nun vollständig in Kubernetes übertragen.

Vorraussetzung:

* Kubernetes-Cluster läuft in Docker for Desktop


Ziel:

* Kubernetes-Konfiguration erstellen
* Deployments für beide Services definieren
* Services für die Kommunikation bereitstellen
* bestehende Architektur in Kubernetes abbilden

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `10. Aufgabe` um.**

---

### Schritt 1 – Kubernetes-Ordner erstellen

Erstellt im Projekt-Root einen neuen Ordner:

```bash
mkdir k8s
```

---

### Schritt 2 – Dateien anlegen

Legt im Ordner `k8s` folgende Dateien an:

```text
k8s/
├── data-service-deployment.yaml
├── data-service-service.yaml
├── ai-service-deployment.yaml
├── ai-service-service.yaml
└── ai-api-secret.yaml
```

---

### Schritt 3 – Data Service Deployment

Datei:

```text
k8s/data-service-deployment.yaml
```

Inhalt:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: data-service
spec:
  replicas: 1
  selector:
    matchLabels:
      app: data-service
  template:
    metadata:
      labels:
        app: data-service
    spec:
      containers:
      - name: data-service
        image: data-service
        ports:
        - containerPort: 8000
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 5
```

### Schritt 4 – Data Service Service

### Schritt 5 – AI Service Deployment

Datei:

```text
k8s/ai-service-deployment.yaml
```

Inhalt:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ai-service
spec:
  replicas: 1
  selector:
    matchLabels:
      app: ai-service
  template:
    metadata:
      labels:
        app: ai-service
    spec:
      containers:
      - name: ai-service
        image: ai-service
        ports:
        - containerPort: 8000
        env:
        - name: OPENAI_API_KEY
          valueFrom:
            secretKeyRef:
              name: ai-api-secret
              key: OPENAI_API_KEY
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 5
```

### Schritt 7 – API-Key in Base64 umwandeln

API-Key in Base64 umwandeln

Linux / Mac:

```bash
echo -n "YOUR_API_KEY" | base64
```

Windows PowerShell:

```powershell
[Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes("YOUR_API_KEY"))
```

Beispiel Ergebnis:

```text
YWJjMTIzNDU2
```

Wichtig:

Base64 ist keine Verschlüsselung, sondern nur Kodierung. Der Wert kann wieder dekodiert werden.

---

### Schritt 8 – Secret YAML erstellen

Datei:

```text
k8s/ai-api-secret.yaml
```

Inhalt:

```yaml 
apiVersion: v1
kind: Secret
metadata:
  name: ai-api-secret
type: Opaque
data:
  OPENAI_API_KEY: YWJjMTIzNDU2
```

Wichtig:

* Der Wert muss Base64-codiert sein
* Key-Name muss exakt übereinstimmen:
  `OPENAI_API_KEY`

---

### Schritt 9 – Deployment starten

Startet alle Ressourcen:

```bash
kubectl apply -f k8s/
```

---

### Schritt 10 – Pods prüfen

```bash
kubectl get pods
```

Es sollten zwei Pods laufen:

```text
NAME                            READY   STATUS    RESTARTS   AGE
ai-service-55cd9dc5b5-5cq6d     1/1     Running   0          17s
data-service-57b586bf47-bcblj   1/1     Running   0          11m
```

**Hinweis:** Es müssen die passenden Images genutzt werden, also die lokal erstellten Images

---

### Schritt 11 – Zugriff testen

Leitet den AI Service lokal weiter:

```bash
kubectl port-forward service/ai-service 8001:8000
```

Test im Browser:

```bash
http://localhost:8001/analysis
```
