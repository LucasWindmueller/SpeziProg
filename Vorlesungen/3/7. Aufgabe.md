## Aufgabe 7 – AI Service mit Docker Compose integrieren

Der AI Service wurde lokal erstellt und ruft den Data Service über `localhost` ab.
Nun wird der AI Service in Docker Compose integriert.

Wichtig:
Innerhalb von Docker Compose funktioniert `localhost` nicht für die Kommunikation zwischen Containern.
Stattdessen wird der Service-Name verwendet:

```text
http://data-service:8000/metrics
```

Ziel:

* AI Service für Docker Compose vorbereiten
* Data Service und AI Service gemeinsam starten
* Kommunikation zwischen Containern herstellen
* Endpoint `/analysis` über Docker Compose testen
* Container mit non-root user ausführen

**Kopiert den bisherigen Projektstand in einen neuen Ordner und benennt diesen in `7. Aufgabe` um.**

---

### Schritt 1 – URL im AI Service anpassen

Öffnet die Datei:

```text
ai-service/main.py
```

Ändert den Request von:

```python
response = requests.get("http://localhost:8000/metrics")
```

zu:

```python
response = requests.get("http://data-service:8000/metrics")
```

Die Datei sieht danach so aus:

```python
from fastapi import FastAPI
import requests

app = FastAPI()

@app.get("/analysis")
def analyze():
    response = requests.get("http://data-service:8000/metrics")
    return response.json()
```

---

### Schritt 2 – Dockerfile für den AI Service erstellen

Erstellt im Ordner `ai-service` eine Datei mit dem Namen:

```text
Dockerfile
```

Inhalt:

```dockerfile
FROM python:3.13-slim

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1

RUN apt-get update && apt-get upgrade -y && rm -rf /var/lib/apt/lists/*

WORKDIR /app

COPY ai-service/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY ai-service/main.py .

RUN useradd -m appuser
USER appuser

EXPOSE 8000

CMD ["python", "-m", "uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Hinweis:

Container sollten aus Sicherheitsgründen nicht als root laufen.
Der non-root user `appuser` stellt sicher, dass der Service mit minimalen Rechten ausgeführt wird.

---

### Schritt 3 – docker-compose.yml erweitern

Öffnet die Datei:

```text
docker-compose.yml
```

Erweitert sie um den `ai-service`:

```yaml
services:
  data-service:
    build:
      context: .
      dockerfile: data-service/Dockerfile
    ports:
      - "8000:8000"

  ai-service:
    build:
      context: .
      dockerfile: ai-service/Dockerfile
    ports:
      - "8001:8000"
```

---

### Schritt 4 – Anwendung starten

Startet beide Services im Projekt-Root:

```bash
docker compose up --build -d
```

---

### Schritt 5 – Container prüfen

```bash
docker compose ps
```

Es sollten zwei Services laufen:

```text
data-service
ai-service
```

---

### Schritt 6 – API testen

Prüft im Browser:

```bash
http://localhost:8001/analysis
```

---

### Beispiel erwartetes Ergebnis

```json
{
  "terms": [
    {
      "name": "coffee",
      "mean": 72.9,
      "peak": 100,
      "trend": "decreasing"
    }
  ]
}
```

---

Wenn der Endpoint `/analysis` erreichbar ist, kommunizieren Data Service und AI Service erfolgreich über Docker Compose.

Zum Beenden:

```bash
docker compose down
```
